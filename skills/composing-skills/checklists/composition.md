# Checklist: composition (before executing a multi-skill task)

- [ ] Capabilities the task needs are **named** (route by need, not phrasing).
- [ ] Each need is mapped to its **most specific** skill.
- [ ] The set is the **smallest** that covers every need (no gap, minimal overlap).
- [ ] No skill in the set will be **<20% used** (else drop or load only its hot slice).
- [ ] Order respects the **dependency graph** (tier N → tier ≤N; plan→execute→verify).
- [ ] `context-budgeting` treated as **ambient**; `verifying-reasoning` placed as a **gate**.
- [ ] Bodies load **sequentially per phase**, not all upfront.
- [ ] Hand-offs are **references**, not copied content; state passes via the **ledger**.
- [ ] Checked for the **monolith smell** (no single skill doing everything).
- [ ] For rare specialists, checked the **registry** rather than bloating the resting menu.
