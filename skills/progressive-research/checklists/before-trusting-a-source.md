# Checklist: before trusting a source

Run this before you act on a researched fact.

- [ ] Is this fact **load-bearing** (I'll act on it / ship it)? If no, a single decent
      source is fine, stop here.
- [ ] Is the source **primary** (spec, official docs, source code) or derivative?
- [ ] Does it state a **version/date**, and does that match my environment?
- [ ] For load-bearing claims: do I have a **second independent source** that agrees?
- [ ] Do any sources **disagree**? If so, surface the disagreement, don't silently pick.
- [ ] Security / destructive / irreversible claim? → corroborate against primary **and**
      hand to `verifying-reasoning`.
- [ ] Have I recorded a **pointer** (url/line) so the claim is auditable later?
- [ ] If unconfirmed after 2-3 cycles: have I labeled it **unverified** and given a next step?
